Rate limiting protects the API from abuse and ensures fair usage across all clients. This guide explains how rate limits work, how to handle them gracefully, and best practices for optimizing your API usage.

## Rate Limit Overview

| Plan       | Requests/Minute | Burst Limit | Batch Limits |
| ---------- | --------------- | ----------- | ------------ |
| Free       | 60              | 10          | 10/batch     |
| Starter    | 300             | 50          | 50/batch     |
| Pro        | 1,000           | 100         | 100/batch    |
| Enterprise | 5,000+          | Custom      | Custom       |

## How Rate Limits Work

### Rate Limit Headers

Every API response includes rate limit information:

```http
HTTP/1.1 200 OK
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 987
X-RateLimit-Reset: 1715412600
X-RateLimit-Retry-After: 0
```

| Header                    | Description                                   |
| ------------------------- | --------------------------------------------- |
| `X-RateLimit-Limit`       | Maximum requests per window                   |
| `X-RateLimit-Remaining`   | Requests remaining in current window          |
| `X-RateLimit-Reset`       | Unix timestamp when the limit resets          |
| `X-RateLimit-Retry-After` | Seconds to wait before retrying (only on 429) |

### Rate Limit Windows

Rate limits are calculated using a sliding window algorithm:

```
Window Size: 60 seconds (1 minute)
┌─────────────────────────────────────┐
│    Sliding Window                   │
│  ┌─────┐                    ┌─────┐ │
│  │ 10s │ 20s  30s  40s  50s │ 60s │ │
│  └─────┘                    └─────┘ │
│     └── Requests count toward limit │
└─────────────────────────────────────┘
```

## Handling Rate Limit Errors

### 429 Response Format

```json
{
  "error": {
    "type": "rate_limit_error",
    "message": "Rate limit exceeded",
    "code": "rate_limit_exceeded",
    "details": {
      "limit": 1000,
      "remaining": 0,
      "reset_at": "2026-05-11T10:31:00Z",
      "retry_after": 45
    }
  }
}
```

### Basic Rate Limit Handling

```javascript
// src/rate-limit/handler.js

class RateLimitHandler {
  constructor(options = {}) {
    this.defaultRetryAfter = options.defaultRetryAfter || 60;
    this.onRateLimit = options.onRateLimit || null;
  }

  /**
   * Check if response indicates rate limiting
   */
  isRateLimitError(response) {
    return response.status === 429;
  }

  /**
   * Extract retry-after value from response or header
   */
  getRetryAfter(response) {
    // Try to get from response body
    if (response.body?.details?.retry_after) {
      return response.body.details.retry_after;
    }

    // Try to get from header
    const headerValue = response.headers.get("X-RateLimit-Retry-After");
    if (headerValue) {
      return parseInt(headerValue, 10);
    }

    // Default fallback
    return this.defaultRetryAfter;
  }

  /**
   * Calculate wait time from reset timestamp
   */
  getWaitTimeFromReset(resetTimestamp) {
    const now = Math.floor(Date.now() / 1000);
    const reset = parseInt(resetTimestamp, 10);

    return Math.max(0, reset - now);
  }
}

/**
 * Sleep utility
 */
function sleep(seconds) {
  return new Promise((resolve) => setTimeout(resolve, seconds * 1000));
}

/**
 * Wait for rate limit to reset
 */
async function waitForRateLimitReset(retryAfter, handler) {
  console.log(`Rate limit reached. Waiting ${retryAfter} seconds...`);

  if (handler.onRateLimit) {
    handler.onRateLimit(retryAfter);
  }

  await sleep(retryAfter);
}
```

### Automatic Retry with Rate Limit Awareness

```javascript
// src/rate-limit/auto-retry.js

import { handleResponse, ZephyrAPIError } from "../errors/handler.js";

class RateLimitAwareClient {
  constructor(apiKey, options = {}) {
    this.apiKey = apiKey;
    this.rateLimitHandler = new RateLimitHandler({
      onRateLimit: (retryAfter) => {
        console.log(`Rate limit alert: waiting ${retryAfter}s`);
        metrics.increment("rate_limit_wait_seconds", retryAfter);
      },
    });
    this.maxRetries = options.maxRetries || 3;
    this.baseDelay = options.baseDelay || 1;
  }

  /**
   * Make request with automatic rate limit handling
   */
  async request(method, endpoint, body = null, retries = 0) {
    const response = await fetch(`${this.baseUrl}${endpoint}`, {
      method,
      headers: this.getHeaders(),
      body: body ? JSON.stringify(body) : null,
    });

    // Handle rate limit
    if (this.rateLimitHandler.isRateLimitError(response)) {
      if (retries >= this.maxRetries) {
        throw new Error("Max retries exceeded due to rate limiting");
      }

      const retryAfter = this.rateLimitHandler.getRetryAfter(response);

      // Add jitter to prevent thundering herd
      const jitter = Math.random() * 0.3 + 1;
      const delay = retryAfter * jitter;

      console.log(`Rate limited. Retrying in ${delay.toFixed(1)}s...`);

      await sleep(delay);
      return this.request(method, endpoint, body, retries + 1);
    }

    return handleResponse(response);
  }
}
```

## Optimizing Request Patterns

### 1. Request Batching

Reduce the number of API calls by batching operations:

```javascript
// Bad: Multiple individual requests
async function createProducts(products) {
  const results = [];
  for (const product of products) {
    const result = await client.post("/products", product);
    results.push(result);
  }
  return results; // N API calls
}

// Good: Single batch request
async function createProducts(products) {
  const result = await client.post("/products/bulk", { products });
  return result.data; // 1 API call
}
```

### 2. Caching Responses

Cache frequently accessed data to reduce API calls:

```javascript
// src/cache/product-cache.js

class ProductCache {
  constructor(ttlSeconds = 300) {
    this.cache = new Map();
    this.ttl = ttlSeconds * 1000;
  }

  /**
   * Get cached product or fetch fresh
   */
  async get(productId) {
    const cached = this.cache.get(productId);

    if (cached && Date.now() - cached.timestamp < this.ttl) {
      return cached.data;
    }

    // Fetch fresh data
    const product = await client.get(`/products/${productId}`);

    this.cache.set(productId, {
      data: product,
      timestamp: Date.now(),
    });

    return product;
  }

  /**
   * Batch fetch with deduplication
   */
  async getMany(productIds) {
    const results = [];
    const toFetch = [];

    // Check cache first
    for (const id of productIds) {
      const cached = this.cache.get(id);
      if (cached && Date.now() - cached.timestamp < this.ttl) {
        results.push({ id, data: cached.data, fromCache: true });
      } else {
        toFetch.push(id);
      }
    }

    // Fetch missing in batches
    if (toFetch.length > 0) {
      const batchSize = 100;
      for (let i = 0; i < toFetch.length; i += batchSize) {
        const batch = toFetch.slice(i, i + batchSize);
        const response = await client.get(
          `/products/batch?ids=${batch.join(",")}`,
        );

        for (const product of response.data) {
          this.cache.set(product.id, {
            data: product,
            timestamp: Date.now(),
          });
          results.push({ id: product.id, data: product, fromCache: false });
        }
      }
    }

    return results;
  }

  /**
   * Invalidate cache entry
   */
  invalidate(productId) {
    this.cache.delete(productId);
  }

  /**
   * Clear all cache
   */
  clear() {
    this.cache.clear();
  }
}

export const productCache = new ProductCache(300); // 5 minute TTL
```

### 3. Pagination Optimization

Fetch only what you need:

```javascript
// Bad: Fetch all pages
async function getAllProducts() {
  const products = [];
  let page = 1;

  while (true) {
    const response = await client.get(`/products?page=${page}&per_page=100`);
    products.push(...response.data);

    if (page >= response.meta.total_pages) break;
    page++;
  }

  return products;
}

// Good: Fetch with limits
async function getActiveProducts(limit = 20) {
  const response = await client.get(
    `/products?status=active&per_page=${limit}`,
  );
  return response.data;
}
```

### 4. Webhooks for Real-time Updates

Use webhooks instead of polling:

```javascript
// Instead of polling for inventory changes...
// Bad: Polling every 30 seconds
setInterval(async () => {
  const products = await client.get("/products?status=active");
  // Check for inventory changes...
}, 30000);

// Good: Subscribe to webhooks
await client.post("/webhooks", {
  url: "https://your-app.com/webhooks",
  events: ["product.inventory_low", "product.inventory_out_of_stock"],
  secret: "whsec_...",
});

// Then handle events in your webhook handler
app.post("/webhooks", (req, res) => {
  const event = req.headers["x-webhook-event"];

  if (event === "product.inventory_low") {
    handleLowInventory(req.body.data);
  }

  res.status(200).send("OK");
});
```

## Rate Limit Monitoring

### Track Rate Limit Usage

```javascript
// src/rate-limit/monitor.js

class RateLimitMonitor {
  constructor() {
    this.callsThisMinute = 0;
    this.resetTime = this.calculateResetTime();
    this.startTime = Date.now();
  }

  calculateResetTime() {
    const now = new Date();
    const nextMinute = new Date(now);
    nextMinute.setSeconds(60, 0);
    return nextMinute.getTime();
  }

  /**
   * Record an API call
   */
  recordCall(limit, remaining) {
    this.callsThisMinute++;
    this.checkReset();

    return {
      callsThisMinute: this.callsThisMinute,
      limit,
      remaining,
      resetIn: Math.ceil((this.resetTime - Date.now()) / 1000),
      usagePercent: (((limit - remaining) / limit) * 100).toFixed(1),
    };
  }

  /**
   * Check if we need to reset the window
   */
  checkReset() {
    if (Date.now() >= this.resetTime) {
      this.callsThisMinute = 0;
      this.resetTime = this.calculateResetTime();
    }
  }

  /**
   * Get current status
   */
  getStatus(limit, remaining) {
    return {
      limit,
      remaining,
      used: limit - remaining,
      usagePercent: (((limit - remaining) / limit) * 100).toFixed(1),
      resetIn: Math.ceil((this.resetTime - Date.now()) / 1000),
      uptime: Date.now() - this.startTime,
    };
  }

  /**
   * Check if we should throttle requests
   */
  shouldThrottle(limit, remaining, bufferPercent = 20) {
    const buffer = limit * (bufferPercent / 100);
    return remaining < buffer;
  }
}

export const rateLimitMonitor = new RateLimitMonitor();
```

## Adaptive Rate Limiting

```javascript
// src/rate-limit/adaptive.js

/**
 * Adaptive client that adjusts request rate based on responses
 */
class AdaptiveRateLimitClient {
  constructor(client) {
    this.client = client;
    this.currentDelay = 0;
    this.minDelay = 0;
    this.maxDelay = 5000;
    this.successStreak = 0;
    this.backoffFactor = 1.1;
    this.boostFactor = 0.9;
  }

  /**
   * Make request with adaptive rate limiting
   */
  async request(method, endpoint, body) {
    // Apply adaptive delay
    if (this.currentDelay > 0) {
      await sleep(this.currentDelay);
    }

    const startTime = Date.now();

    try {
      const response = await this.client.request(method, endpoint, body);

      // Success - potentially decrease delay
      if (response.status === 200) {
        this.successStreak++;
        if (this.successStreak > 10) {
          this.currentDelay = Math.max(
            this.minDelay,
            this.currentDelay * this.boostFactor,
          );
          this.successStreak = 0;
        }
      }

      return response;
    } catch (error) {
      if (error.code === "rate_limit_exceeded") {
        // Increase delay
        this.currentDelay = Math.min(
          this.maxDelay,
          this.currentDelay * this.backoffFactor + 100,
        );
        this.successStreak = 0;
      }

      throw error;
    }
  }

  /**
   * Get current rate limit configuration
   */
  getRateLimitStatus() {
    return {
      currentDelay: this.currentDelay,
      estimatedRequestsPerMinute:
        this.currentDelay > 0
          ? Math.round(60000 / this.currentDelay)
          : "unlimited",
    };
  }
}
```

## Best Practices

<Callout type="tip">**Rate Limit Optimization Tips:**</Callout>

1. **Use webhooks instead of polling** - Real-time updates use fewer API calls
2. **Cache aggressively** - Cache product and customer data
3. **Batch operations** - Use bulk APIs when available
4. **Implement exponential backoff** - Don't hammer the API on errors
5. **Monitor your usage** - Track rate limit headers on every response
6. **Request only what you need** - Use filtering and pagination
7. **Queue non-urgent requests** - Process analytics and syncs during off-peak

## Request Prioritization

For high-volume applications, prioritize requests:

```javascript
// src/rate-limit/priority-queue.js

class PriorityQueue {
  constructor(client, options = {}) {
    this.client = client;
    this.maxConcurrent = options.maxConcurrent || 5;
    this.queue = [];
    this.running = 0;
  }

  /**
   * Add request to queue
   */
  enqueue(method, endpoint, body, priority = 5) {
    return new Promise((resolve, reject) => {
      this.queue.push({
        method,
        endpoint,
        body,
        priority, // 1 = highest, 10 = lowest
        resolve,
        reject,
      });

      // Sort by priority
      this.queue.sort((a, b) => a.priority - b.priority);

      this.processQueue();
    });
  }

  /**
   * Process queue with concurrency limit
   */
  async processQueue() {
    while (this.queue.length > 0 && this.running < this.maxConcurrent) {
      const request = this.queue.shift();
      this.running++;

      try {
        const result = await this.client.request(
          request.method,
          request.endpoint,
          request.body,
        );
        request.resolve(result);
      } catch (error) {
        request.reject(error);
      } finally {
        this.running--;
        this.processQueue();
      }
    }
  }
}

// Usage: High priority for checkout, low for analytics sync
const queue = new PriorityQueue(client, { maxConcurrent: 3 });

// High priority - customer checkout
queue.enqueue("POST", "/orders", orderData, 1);

// Low priority - analytics sync
queue.enqueue("POST", "/analytics/events/batch", events, 10);
```
