Proper environment configuration is critical for building reliable integrations. This guide covers strategies for managing different environments, configuration patterns, and best practices for keeping your API keys and secrets secure.

## Environment Overview

| Environment     | Purpose                       | API Base URL                        | Characteristics                   |
| --------------- | ----------------------------- | ----------------------------------- | --------------------------------- |
| **Development** | Local development and testing | `https://api-sandbox.zephyr.com/v1` | Full API access, relaxed limits   |
| **Staging**     | Pre-production testing        | `https://api-staging.zephyr.com/v1` | Production-like, real data subset |
| **Production**  | Live applications             | `https://api.zephyr.com/v1`         | Full rate limits, real data       |

## Configuration Patterns

### 1. Environment Variables (Recommended)

The most common and secure approach for server-side applications:

```bash
# .env.development
ZEPHYR_API_KEY=sk_test_dev_abc123...
ZEPHYR_API_URL=https://api-sandbox.zephyr.com/v1
ZEPHYR_WEBHOOK_SECRET=whsec_test_dev_xyz789...
LOG_LEVEL=debug

# .env.staging
ZEPHYR_API_KEY=sk_test_staging_abc123...
ZEPHYR_API_URL=https://api-staging.zephyr.com/v1
ZEPHYR_WEBHOOK_SECRET=whsec_staging_xyz789...
LOG_LEVEL=info

# .env.production
ZEPHYR_API_KEY=sk_live_prod_abc123...
ZEPHYR_API_URL=https://api.zephyr.com/v1
ZEPHYR_WEBHOOK_SECRET=whsec_live_xyz789...
LOG_LEVEL=warn
```

```javascript
// src/config.js
import dotenv from "dotenv";
import path from "path";

// Load environment-specific .env file
const env = process.env.NODE_ENV || "development";
dotenv.config({ path: path.resolve(`.env.${env}`) });

export const config = {
  api: {
    key: process.env.ZEPHYR_API_KEY,
    url: process.env.ZEPHYR_API_URL,
    webhookSecret: process.env.ZEPHYR_WEBHOOK_SECRET,
  },
  logging: {
    level: process.env.LOG_LEVEL || "info",
  },
  features: {
    enableMetrics: process.env.ENABLE_METRICS === "true",
    enableCaching: process.env.ENABLE_CACHING !== "false",
  },
};

// Validate required configuration
function validateConfig() {
  const required = ["api.key", "api.url"];
  const missing = required.filter((path) => {
    const value = path.split(".").reduce((obj, key) => obj?.[key], config);
    return !value;
  });

  if (missing.length > 0) {
    throw new Error(`Missing required configuration: ${missing.join(", ")}`);
  }
}

validateConfig();
```

### 3. Secret Management Services

For production deployments, use dedicated secret management:

```javascript
// src/config/secrets.js - AWS Secrets Manager example
import {
  SecretsManagerClient,
  GetSecretValueCommand,
} from "@aws-sdk/client-secrets-manager";

const secretsClient = new SecretsManagerClient({ region: "us-east-1" });

async function getSecret(secretName) {
  try {
    const command = new GetSecretValueCommand({ SecretId: secretName });
    const response = await secretsClient.send(command);
    return JSON.parse(response.SecretString);
  } catch (error) {
    console.error(`Failed to fetch secret ${secretName}:`, error);
    throw error;
  }
}

export async function loadProductionConfig() {
  const secrets = await getSecret("zephyr/production");

  return {
    apiKey: secrets.ZEPHYR_API_KEY,
    webhookSecret: secrets.ZEPHYR_WEBHOOK_SECRET,
    databaseUrl: secrets.DATABASE_URL,
    // ... other secrets
  };
}
```

### Rotating API Keys

```javascript
// src/services/key-rotation.js
import { client } from "../client.js";

class KeyRotationService {
  constructor(oldKey, newKey) {
    this.oldKey = oldKey;
    this.newKey = newKey;
    this.client = new ZephyrClient(newKey);
  }

  /**
   * Rotate API key with zero downtime
   * Steps: Create new key -> Update services -> Verify -> Revoke old key
   */
  async rotate(gracePeriodHours = 24) {
    console.log("Starting API key rotation...");

    // Step 1: Verify new key works
    console.log("Verifying new key...");
    const testResult = await this.client.get("/products?per_page=1");
    console.log("New key verified successfully");

    // Step 2: Log rotation timestamp
    const rotationRecord = {
      timestamp: new Date().toISOString(),
      oldKeyPrefix: this.oldKey.substring(0, 10) + "...",
      newKeyPrefix: this.newKey.substring(0, 10) + "...",
      gracePeriodHours,
    };

    // In production, store this in a secure audit log
    console.log("Key rotation record:", rotationRecord);

    // Step 3: Schedule old key revocation (after grace period)
    this.scheduleRevocation(gracePeriodHours);

    return rotationRecord;
  }

  scheduleRevocation(hours) {
    // In production, use a job scheduler
    console.log(`
      Old key will be revoked in ${hours} hours.
      Before then, you can rollback by updating ZEPHYR_API_KEY back to the old key.
    `);

    // Example: Schedule with node-cron
    // cron.schedule(`*/${hours} * * *`, () => this.revokeOldKey());
  }

  async revokeOldKey() {
    try {
      // Revoke the old key via API
      const oldClient = new ZephyrClient(this.oldKey);
      await oldClient.delete("/api-keys/self");
      console.log("Old key revoked successfully");
    } catch (error) {
      console.error("Failed to revoke old key:", error.message);
      // Alert administrators
    }
  }
}
```

## Managing Multiple Environments

### Environment Detection

```javascript
// src/environment.js
export function getEnvironment() {
  // Check explicit environment variable first
  if (process.env.ZEPHYR_ENV) {
    return process.env.ZEPHYR_ENV;
  }

  // Detect from hostname or URL
  const hostname = process.env.HOSTNAME || "";

  if (hostname.includes("localhost") || hostname.includes("127.0.0.1")) {
    return "development";
  }

  if (hostname.includes("staging") || hostname.includes("test")) {
    return "staging";
  }

  if (hostname.includes("production") || hostname.includes("prod")) {
    return "production";
  }

  // Default to development
  return "development";
}

export function isProduction() {
  return getEnvironment() === "production";
}

export function isDevelopment() {
  return getEnvironment() === "development";
}
```

### Environment-Specific Code

```javascript
// src/services/analytics.js
import { isProduction } from "../environment.js";

export class AnalyticsService {
  async trackPurchase(order) {
    const event = {
      event_name: "purchase_completed",
      user_id: order.customer.email,
      properties: {
        order_id: order.id,
        revenue: order.pricing.grand_total,
      },
    };

    // In development, log events instead of sending
    if (!isProduction()) {
      console.log("[Analytics] Would track:", event);
      return { simulated: true };
    }

    return this.client.analytics.track(event);
  }
}
```

```javascript
      const product = await client.post("/products", {
```

## Best Practices Checklist

<Callout type="tip">**Security Checklist:**</Callout>

- [ ] Never commit API keys to version control
- [ ] Use different keys for each environment
- [ ] Implement principle of least privilege for key permissions
- [ ] Enable IP whitelisting in production
- [ ] Set key expiration dates
- [ ] Implement key rotation every 90 days
- [ ] Use secret management services in production
- [ ] Validate all configuration on startup
- [ ] Log configuration loading (without secrets)
- [ ] Separate webhook secrets from API keys

## Environment Migration

When migrating between environments:

1. **Backup current configuration**
2. **Create new environment credentials**
3. **Update DNS/routing if needed**
4. **Deploy with new configuration**
5. **Verify integration works**
6. **Monitor for 24-48 hours**
7. **Decommission old environment**
