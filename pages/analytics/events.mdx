Track user behavior and product interactions with our event-driven analytics API. Ingest millions of events per day with sub-second query latency.

## Event Model

Every event follows our unified schema:

```typescript
interface AnalyticsEvent {
  event_id: string; // Unique event identifier
  event_name: string; // e.g., "page_view", "purchase"
  timestamp: string; // ISO 8601 timestamp
  user_id?: string; // User identifier (if known)
  anonymous_id?: string; // Anonymous visitor ID
  session_id: string; // Browser/session identifier
  properties: Record<string, any>; // Event-specific data
  context: {
    ip: string;
    user_agent: string;
    locale: string;
    page_url: string;
    referrer?: string;
  };
}
```

## Sending Events

### Server-side Event Tracking

```python
# Python SDK
from zephyr import ZephyrClient

client = ZephyrClient(api_key='sk_live_abc123...')

# Track a single event
client.analytics.track(
    event_name="product_purchased",
    user_id="usr_abc123",
    properties={
        "product_id": "prod_xyz789",
        "product_name": "Premium Wireless Headphones",
        "price": 199.99,
        "currency": "USD",
        "quantity": 1,
        "payment_method": "credit_card"
    }
)
```

```javascript
// Node.js SDK
import { ZephyrClient } from "@zephyr/node-sdk";

const client = new ZephyrClient({ apiKey: "sk_live_abc123..." });
```

## Querying Events

### List Events

```bash
curl -X GET "https://api.zephyr.com/v1/analytics/events?page=1&per_page=100" \
  -H "Authorization: Bearer sk_live_abc123..."
```

**Query Parameters:**

| Parameter    | Type    | Required | Description                            |
| ------------ | ------- | -------- | -------------------------------------- |
| `page`       | integer | No       | Page number (default: 1)               |
| `per_page`   | integer | No       | Items per page (default: 20, max: 100) |
| `event_name` | string  | No       | Filter by event type                   |
| `user_id`    | string  | No       | Filter by user                         |
| `start_date` | string  | No       | Start of date range (ISO 8601)         |
| `end_date`   | string  | No       | End of date range (ISO 8601)           |
| `sort`       | string  | No       | Sort field (`timestamp`, `event_name`) |
| `order`      | string  | No       | Sort order (`asc`, `desc`)             |

## Core Event Types

### Page Events

| Event               | Description                | Required Properties        | Example Properties         |
| ------------------- | -------------------------- | -------------------------- | -------------------------- |
| `page_view`         | User viewed a page         | `page_url`, `page_name`    | `referrer`, `scroll_depth` |
| `page_scroll`       | User scrolled to bottom    | `page_url`, `scroll_depth` | `time_on_page`             |
| `search_performed`  | User searched              | `search_term`              | `results_count`            |
| `search_no_results` | Search returned no results | `search_term`              | -                          |

### User Events

| Event                      | Description           | Required Properties         | Example Properties |
| -------------------------- | --------------------- | --------------------------- | ------------------ |
| `user_signed_up`           | New user registration | `user_id`, `method`         | `plan`, `source`   |
| `user_logged_in`           | User login            | `user_id`, `method`         | `remember_me`      |
| `user_logged_out`          | User logout           | `user_id`                   | -                  |
| `user_updated_profile`     | Profile change        | `user_id`, `fields_changed` | `previous_values`  |
| `password_reset_requested` | Password reset        | `user_id`, `method`         | -                  |

### E-Commerce Events

| Event                       | Description         | Required Properties          | Example Properties          |
| --------------------------- | ------------------- | ---------------------------- | --------------------------- |
| `product_viewed`            | Product page viewed | `product_id`, `product_name` | `price`, `category`         |
| `product_added_to_cart`     | Item added to cart  | `product_id`, `quantity`     | `price`, `cart_value`       |
| `product_removed_from_cart` | Item removed        | `product_id`                 | `cart_value`                |
| `cart_abandoned`            | Cart abandoned      | -                            | `cart_value`, `items_count` |
| `checkout_started`          | Checkout initiated  | `cart_value`, `item_count`   | `coupon_code`               |
| `checkout_step_completed`   | Checkout step done  | `step_name`                  | `step_number`               |
| `purchase_completed`        | Order placed        | `order_id`, `revenue`        | `payment_method`, `items`   |
| `refund_initiated`          | Refund started      | `order_id`, `amount`         | `reason`                    |
| `refund_completed`          | Refund processed    | `order_id`, `amount`         | `reason`                    |

## Event Validation

### Required Fields

| Field        | Type   | Required | Description                                          |
| ------------ | ------ | -------- | ---------------------------------------------------- |
| `event_name` | string | Yes      | Must be 1-128 characters, alphanumeric + underscores |
| `timestamp`  | string | Yes      | ISO 8601 format                                      |
| `properties` | object | Yes      | At minimum, an empty object `{}`                     |

### Property Constraints

| Property Type    | Max Length | Notes                              |
| ---------------- | ---------- | ---------------------------------- |
| `event_name`     | 128 chars  | Alphanumeric + underscore          |
| `user_id`        | 256 chars  | Alphanumeric + underscore + hyphen |
| `property key`   | 128 chars  | Alphanumeric + underscore          |
| `property value` | 1024 chars | Strings are truncated              |
| `array items`    | 1000 items | Arrays are truncated               |
| `object depth`   | 10 levels  | Objects are flattened              |

### Validation Errors

```json
{
  "error": {
    "type": "validation_error",
    "message": "Event validation failed",
    "code": "invalid_event",
    "details": {
      "errors": [
        {
          "field": "event_name",
          "value": "invalid event name!",
          "message": "Event name can only contain alphanumeric characters and underscores"
        },
        {
          "field": "timestamp",
          "value": "not-a-date",
          "message": "Invalid ISO 8601 timestamp"
        }
      ]
    },
    "request_id": "req_abc123xyz"
  }
}
```

## SDK Configuration

### Node.js SDK

```javascript
import { ZephyrAnalytics } from "@zephyr/analytics-sdk";

const analytics = new ZephyrAnalytics({
  apiKey: process.env.ZEPHYR_ANALYTICS_KEY,

  // Batch settings
  batch: {
    enabled: true,
    size: 20, // Flush after N events
    flushInterval: 5000, // Or every N milliseconds
    maxRetries: 3,
  },

  // Retry settings
  retry: {
    enabled: true,
    maxRetries: 3,
    initialDelay: 1000,
  },

  // Logging
  debug: process.env.NODE_ENV === "development",
});

// Wrap your Express app
analytics.connect(app);
```

```python
from zephyr.analytics import Analytics
```

### Handling Event Failures

```javascript
// Node.js - Handle batch failures
try {
  const result = await client.analytics.trackBatch(events);

  if (result.failed > 0) {
    console.error(`${result.failed} events failed:`);
    for (const error of result.errors) {
      console.error(`  - Event ${error.event_index}: ${error.message}`);
    }

    // Retry failed events
    const failedEvents = events.filter((_, i) =>
      result.errors.some((e) => e.event_index === i),
    );
    // ... retry logic
  }
} catch (error) {
  // Network or other errors - retry the entire batch
  await retryBatch(events);
}
```
