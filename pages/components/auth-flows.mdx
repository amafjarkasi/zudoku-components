### Authorization URL Format

```
https://auth.zu.com/oauth/authorize?
  client_id=YOUR_CLIENT_ID&
  redirect_uri=https://yourapp.com/callback&
  response_type=code&
  scope=ecommerce:read analytics:write
```

### Token Exchange Request

```bash
curl -X POST https://auth.zu.com/oauth/token \
  -d "grant_type=authorization_code" \
  -d "code=AUTH_CODE" \
  -d "redirect_uri=https://yourapp.com/callback" \
  -d "client_id=YOUR_CLIENT_ID" \
  -d "client_secret=YOUR_CLIENT_SECRET"
```

### Token Response

```json
{
  "access_token": "zk_access_xyz789...",
  "refresh_token": "zk_refresh_abc456...",
  "expires_in": 3600,
  "token_type": "Bearer",
  "scope": "ecommerce:read analytics:write"
}
```

<Callout type="info">
**JWT vs API Keys:** JWTs are user-scoped and expire automatically. API keys are static and server-scoped. Choose based on whether you need user context or just API access.
</Callout>
