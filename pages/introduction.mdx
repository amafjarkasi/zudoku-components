## Quick Start

Get up and running in 5 minutes:

### Step 1: Create an Account

Visit [dashboard.zephyr.com/signup](https://dashboard.zephyr.com/signup) and sign up using your email or social accounts.

### Step 2: Get Your API Keys

1. Navigate to **Settings > API Keys** in your dashboard
2. Click **Generate New Key**
3. Give it a descriptive name (e.g., "Development Key")
4. **Copy the key immediately** - you won't see it again!

```bash
# Your API key looks like this:
sk_live_abc123xyz789def456...
```

<Callout type="info">
  **Security Warning:** Never commit your API keys to version control. Use environment variables or secret management systems to store them securely.
</Callout>

### Step 3: Make Your First Request

Test your API key with a simple request to list products:

```bash
curl -X GET https://api.zephyr.com/v1/products \
  -H "Authorization: Bearer sk_live_abc123xyz789def456..." \
  -H "Content-Type: application/json"
```

**Expected response:**

```json
{
  "data": [
    {
      "id": "prod_abc123",
      "name": "Premium Wireless Headphones",
      "description": "High-quality Bluetooth headphones with noise cancellation",
      "price": 199.99,
      "currency": "USD",
      "inventory_count": 150,
      "created_at": "2026-05-10T00:00:00Z",
      "updated_at": "2026-05-10T00:00:00Z"
    }
  ],
  "meta": {
    "total": 1,
    "page": 1,
    "per_page": 20,
    "total_pages": 1
  }
}
```

### Step 4: Install an SDK

Choose your language and install our official SDK:

```bash
# Node.js / JavaScript
npm install @zephyr/node-sdk

# Python
pip install zephyr

# React
npm install @zephyr/react-sdk

# Go
go get github.com/zephyr/go-sdk

# Ruby
gem install zephyr
```

### Step 5: Initialize and Make Your First Call

```javascript
// Node.js / JavaScript
import { ZephyrClient } from '@zephyr/node-sdk';

const client = new ZephyrClient({
  apiKey: 'sk_live_abc123xyz789def456...'
});

// List products
const products = await client.products.list({ limit: 10 });
console.log(products);
```

```python
# Python
from zephyr import ZephyrClient

client = ZephyrClient(api_key='sk_live_abc123xyz789def456...')

# List products
products = client.products.list(limit=10)
print(products)
```

```tsx
// React
import { useProducts } from '@zephyr/react-sdk';

function ProductList() {
  const { data: products, isLoading, error } = useProducts({ limit: 10 });

  if (isLoading) return <div>Loading...</div>;
  if (error) return <div>Error: {error.message}</div>;

  return (
    <ul>
      {products.map(product => (
        <li key={product.id}>{product.name} - ${product.price}</li>
      ))}
    </ul>
  );
}
```

## Error Handling

Errors are returned with appropriate HTTP status codes and detailed information:

```json
{
  "error": {
    "type": "invalid_request_error",
    "message": "Invalid API key provided",
    "code": "invalid_api_key",
    "details": {
      "field": "Authorization",
      "reason": "API key not found or expired"
    },
    "request_id": "req_abc123xyz",
    "timestamp": "2026-05-10T15:30:00Z"
  }
}
```

Common error codes:
- `invalid_api_key` - Invalid or expired API key
- `rate_limit_exceeded` - Too many requests
- `insufficient_permissions` - API key lacks required permissions
- `resource_not_found` - Requested resource doesn't exist
- `validation_error` - Invalid request parameters
- `server_error` - Internal server error (retry with exponential backoff)

See our [Error Handling Guide](/advanced/errors) for more details.

## Rate Limiting

API requests are rate-limited to ensure fair usage and system stability:

| Plan | Requests/Minute | Requests/Hour | Burst Allowance |
|------|-----------------|---------------|-----------------|
| Free | 10 | 100 | 20 |
| Pro | 100 | 1,000 | 200 |
| Enterprise | Custom | Custom | Custom |

Rate limit headers are included in every response:

```http
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1715347600
```

When rate limits are exceeded, you'll receive a `429 Too Many Requests` response with a `Retry-After` header.

See our [Rate Limiting Guide](/advanced/rate-limiting) for best practices on handling rate limits.

## Web Versioning

We follow semantic versioning for our API: `MAJOR.MINOR.PATCH`

- **MAJOR**: Breaking changes (rare, announced well in advance)
- **MINOR**: New features, backward-compatible
- **PATCH**: Bug fixes and minor improvements

Current stable version: **v2.5.0**

Always specify the API version in your requests:

```bash
# Using the Accept header
curl -X GET https://api.zephyr.com/v1/products \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Accept: application/vnd.zephyr.v2.5+json"

# Or use the version prefix
curl -X GET https://api.zephyr.com/v2.5/products \
  -H "Authorization: Bearer YOUR_API_KEY"
```
