The Orders API handles the complete order lifecycle from cart creation to delivery. Process thousands of orders per minute with automatic inventory deduction, tax calculation, payment processing, and fulfillment routing.

## Order Lifecycle

Orders flow through a defined lifecycle from creation to completion:

<Mermaid chart={`
stateDiagram-v2
    [*] --> CREATED
    CREATED --> PENDING: Payment Initiated
    PENDING --> PAID: Payment Confirmed
    PAID --> PROCESSING: Fulfillment Queued
    PROCESSING --> SHIPPED: Carrier Pickup
    SHIPPED --> DELIVERED: Completed
    
    CREATED --> CANCELLED
    PENDING --> CANCELLED
    PAID --> CANCELLED
    PROCESSING --> CANCELLED
    
    PENDING --> EXPIRED: Timeout
    EXPIRED --> CANCELLED
    
    DELIVERED --> RETURNED
    RETURNED --> REFUNDED
    PAID --> REFUNDED
`} />

### Order Status Details

| Status | Description | Trigger | Can Transition To |
|--------|-------------|---------|-------------------|
| `created` | Order created, awaiting payment | Customer completes checkout | `pending`, `cancelled` |
| `pending` | Payment pending | Payment initiated | `paid`, `cancelled`, `expired` |
| `paid` | Payment confirmed | Payment successful | `processing`, `cancelled` |
| `processing` | Order queued for fulfillment | Payment verified | `shipped`, `cancelled` |
| `shipped` | Order shipped, in transit | Carrier pickup | `delivered` |
| `delivered` | Order delivered successfully | Delivery confirmation | `returned` |
| `cancelled` | Order cancelled | Customer/admin action | None (terminal) |
| `refunded` | Payment refunded | Refund processed | None (terminal) |
| `returned` | Items returned to warehouse | Return received | `refunded` |
| `expired` | Payment window expired | Timeout (usually 15 min) | `cancelled` |

## API Endpoints

### Create Order

Create a new order from checkout or directly.

```bash
POST /v1/orders
```

**Request Body:**

```json
{
  "customer_id": "cust_def456uvw123",
  "email": "sarah.chen@example.com",
  "items": [
    {
      "variant_id": "var_shirt_red_m",
      "quantity": 2,
      "unit_price": 29.99
    }
  ],
  "billing_address": {
    "first_name": "Sarah",
    "last_name": "Chen",
    "line1": "742 Evergreen Terrace",
    "city": "Portland",
    "state": "OR",
    "postal_code": "97201",
    "country": "US"
  },
  "shipping_address": {
    "first_name": "Sarah",
    "last_name": "Chen",
    "line1": "742 Evergreen Terrace",
    "city": "Portland",
    "state": "OR",
    "postal_code": "97201",
    "country": "US"
  },
  "shipping_method": "standard",
  "coupon_code": "SUMMER20",
  "metadata": {
    "source": "web"
  }

```

**Example Request:**

```bash
curl -X POST "https://api.zephyr.com/v1/orders" \
  -H "Authorization: Bearer sk_live_abc123..." \
  -H "Content-Type: application/json" \
  -d @order.json
```

**Response:**

```json

  "order_number": "ORD-2026-001234",
  "status": "created",
  "total": 53.98,
  "created_at": "2026-05-10T15:42:00Z"
}
```

### Get Order

Retrieve a single order by ID or order number.

```bash
GET /v1/orders/{id}
```

**Path Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `id` | string | Yes | Order ID or order number |

**Example Request:**

```bash
curl -X GET "https://api.zephyr.com/v1/orders/ord_abc123xyz789" \
  -H "Authorization: Bearer sk_live_abc123..." \
  -H "Content-Type: application/json"
```

### List Orders

Retrieve a paginated list of orders with filtering.

```bash
GET /v1/orders
```

**Query Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `page` | integer | No | Page number (default: 1) |
| `per_page` | integer | No | Items per page (default: 20, max: 100) |
| `status` | string | No | Filter by status |
| `customer_id` | string | No | Filter by customer ID |
| `email` | string | No | Filter by customer email |
| `created_after` | string | No | ISO 8601 date |
| `created_before` | string | No | ISO 8601 date |
| `sort` | string | No | Sort field (`created_at`, `total`, `updated_at`) |
| `order` | string | No | Sort order (`asc`, `desc`) |

**Example Request:**

```bash
curl -X GET "https://api.zephyr.com/v1/orders?status=shipped&page=1&per_page=20" \
  -H "Authorization: Bearer sk_live_abc123..." \
  -H "Content-Type: application/json"
```

## Order Analytics

Track key order metrics:

| Metric | Formula | Description |
|--------|---------|-------------|
| **Average Order Value (AOV)** | `total_revenue / number_of_orders` | Average amount spent per order |
| **Order Fulfillment Rate** | `orders_delivered / orders_placed * 100` | Percentage of orders successfully delivered |
| **Cancellation Rate** | `orders_cancelled / orders_placed * 100` | Percentage of orders cancelled |
| **Return Rate** | `orders_returned / orders_delivered * 100` | Percentage of delivered orders returned |
| **Refund Rate** | `total_refunded / total_revenue * 100` | Percentage of revenue refunded |
| **Repeat Customer Rate** | `customers_with_multiple_orders / total_customers * 100` | Customer loyalty metric |

## Best Practices

### 1. Implement Idempotency

Use idempotency keys to prevent duplicate orders:

```bash
curl -X POST "https://api.zephyr.com/v1/orders" \
  -H "Authorization: Bearer sk_live_abc123..." \
  -H "Idempotency-Key: 12345678-1234-1234-1234-123456789abc" \
  -H "Content-Type: application/json" \
  -d @order.json
```

```javascript
// Generate idempotency key
const idempotencyKey = crypto.randomUUID();

// Store in localStorage to handle retries
localStorage.setItem('lastOrderKey', idempotencyKey);

// Use in API call
const response = await fetch('https://api.zephyr.com/v1/orders', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${apiKey}`,
    'Idempotency-Key': idempotencyKey,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify(orderData)
});
```

### 2. Handle Webhooks

Set up webhooks for real-time order updates:

```bash

  -H "Authorization: Bearer sk_live_abc123..." \
  -H "Content-Type: application/json" \

    "url": "https://your-app.com/webhooks/orders",
    "events": [

      "order.paid",
      "order.shipped",
      "order.delivered",
      "order.cancelled"
    ],
```

```javascript
// Handle order webhooks
app.post('/webhooks/orders', (req, res) => {
  const signature = req.headers['x-webhook-signature'];
  const payload = req.rawBody;

  // Verify signature
  if (!verifyWebhookSignature(payload, signature, webhookSecret)) {
    return res.status(401).send('Invalid signature');
  }

  const event = req.headers['x-webhook-event'];
  const order = req.body.data;

  switch (event) {
    case 'order.created':
      sendOrderConfirmationEmail(order);
      break;
    case 'order.paid':
      initiateFulfillment(order);
      break;
    case 'order.shipped':
      sendShippingNotification(order);
      break;
    case 'order.delivered':
      requestProductReview(order);
      break;
    case 'order.cancelled':
      processCancellation(order);
      break;
  }

  res.status(200).send('OK');
});
```

### 3. Implement Retry Logic

Handle transient errors with exponential backoff:

```javascript
async function createOrder(orderData, maxRetries = 3) {
  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      const response = await fetch('https://api.zephyr.com/v1/orders', {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${apiKey}`,
          'Content-Type': 'application/json',
          'Idempotency-Key': orderData.idempotencyKey
        },
        body: JSON.stringify(orderData)
      });

      if (response.ok) {
        return await response.json();
      }

      const error = await response.json();

      // Don't retry on validation errors
      if (error.error.code === 'validation_error') {
        throw error;
      }

      // Retry on rate limits or server errors
      if (response.status === 429 || response.status >= 500) {
        const delay = Math.pow(2, attempt) * 1000; // Exponential backoff
        await new Promise(resolve => setTimeout(resolve, delay));
        continue;
      }

      throw error;

    } catch (error) {
      if (attempt === maxRetries) {
        throw error;
      }
      console.log(`Attempt ${attempt} failed, retrying...`);
    }
  }
}
```

### 4. Validate Inventory Before Creating Order

Check inventory before allowing checkout:

```javascript
async function validateCartItems(cartItems) {
  const variantIds = cartItems.map(item => item.variant_id);

  const response = await fetch(
    `https://api.zephyr.com/v1/variants?ids=${variantIds.join(',')}`,
    {
      headers: { 'Authorization': `Bearer ${apiKey}` }
    }
  );

  const variants = await response.json();

  const outOfStockItems = [];

  cartItems.forEach(item => {
    const variant = variants.data.find(v => v.id === item.variant_id);
    if (!variant || variant.inventory_quantity < item.quantity) {
      outOfStockItems.push({
        item: item,
        available: variant ? variant.inventory_quantity : 0
      });
    }
  });

  if (outOfStockItems.length > 0) {
    throw new Error('Some items are out of stock', {
      cause: outOfStockItems
    });
  }

  return true;
}
```

### 5. Implement Order Search

Build efficient order search for your dashboard:

```javascript
async function searchOrders(filters) {
  const params = new URLSearchParams();
```

## Error Handling

### Common Order Errors

| Error Code | Description | Solution |
|------------|-------------|----------|
| `order_not_found` | Order doesn't exist | Verify order ID |
| `invalid_status` | Invalid status transition | Check current status |
| `inventory_insufficient` | Not enough inventory | Update inventory or deny order |
| `payment_failed` | Payment processing failed | Check payment method |
| `address_invalid` | Invalid shipping address | Verify address format |
| `coupon_expired` | Coupon has expired | Use valid coupon |
| `order_locked` | Order cannot be modified | Order is in processing |
