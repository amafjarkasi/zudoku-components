The Products API is the core of Zephyr's e-commerce engine. It allows you to build a comprehensive product catalog, manage inventory, handle variants, and sync product data across multiple channels.

## Overview

A `Product` in our system is more than just a name and price. It's a complex entity that supports:

- **Multiple variants** (size, color, material, etc.)
- **Dynamic pricing tiers** (base price, sale price, bulk pricing)
- **Regional availability** and localization
- **Rich media** (images, videos, documents)
- **Custom attributes** and metadata
- **Inventory tracking** across multiple warehouses
- **Multi-channel synchronization** (web, mobile, marketplace)

### Product Types

| Type | Description | Example |
|------|-------------|---------|
| `physical` | Tangible products that require shipping | Clothing, electronics, furniture |
| `digital` | Downloadable or streamable products | E-books, software, music |
| `service` | Time-based or appointment-based services | Consulting, appointments |
| `gift_card` | Prepaid gift cards with fixed value | $50 gift card |
| `bundle` | Collection of products sold together | Product bundle with discount |

### Product Status

| Status | Description | API Behavior |
|--------|-------------|--------------|
| `draft` | Not visible to customers | Hidden from all queries |
| `active` | Visible and purchasable | Available in queries and cart |
| `archived` | Not purchasable but visible | Read-only, cannot be purchased |
| `deleted` | Soft deleted | Excluded from all queries |

## API Endpoints

### List Products

Retrieve a paginated list of products with optional filtering.

```bash
GET /v1/products
```

**Query Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `page` | integer | No | Page number (default: 1) |
| `per_page` | integer | No | Items per page (default: 20, max: 100) |
| `status` | string | No | Filter by status (`draft`, `active`, `archived`) |
| `category` | string | No | Filter by category slug |
| `tag` | string | No | Filter by tag |
| `search` | string | No | Search in name, description, SKU |
| `sort` | string | No | Sort field (`created_at`, `name`, `price`) |
| `order` | string | No | Sort order (`asc`, `desc`) |
| `since_id` | string | No | Only return products created after this ID |
| `ids` | array | No | Filter by specific product IDs |

**Example Request:**

```bash
curl -X GET "https://api.zephyr.com/v1/products?page=1&per_page=20&status=active&sort=created_at&order=desc" \
  -H "Authorization: Bearer sk_live_abc123..." \
  -H "Content-Type: application/json"
```

**Example Response:**

```json
{
  "data": [
    {
      "id": "prod_abc123",
      "name": "Premium Wireless Headphones",
      "price": 199.99,
      "status": "active",
      "created_at": "2026-05-01T00:00:00Z"
    }
  ],
  "meta": {
    "total": 1250,
    "page": 1,
    "per_page": 20,
    "total_pages": 63
  }
}
```

### Update Product

Update an existing product.

```bash
PUT /v1/products/{id}
```

**Request Body:** Same as create product (all fields are optional).

**Example Request:**

```bash
curl -X PUT "https://api.zephyr.com/v1/products/prod_abc123" \
  -H "Authorization: Bearer sk_live_abc123..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Premium Wireless Headphones Pro",
    "price": 219.99

```

### Delete Product

## Inventory Management

### Inventory Policies

| Policy | Description | Behavior |
|--------|-------------|----------|
| `continue` | Allow backorders | Customers can order even if out of stock |
| `deny` | Stop selling when out of stock | Hide or disable purchase button |
| `track` | Track inventory only | Allow purchase but don't limit quantity |

## Best Practices

### 1. Pagination

Use cursor-based pagination when syncing large catalogs:

```bash
curl -X GET "https://api.zephyr.com/v1/products?per_page=100&page=1" \
  -H "Authorization: Bearer sk_live_abc123..."
```

```javascript
// Recursive function to fetch all products
async function fetchAllProducts(page = 1, products = []) {
  const response = await fetch(
    `https://api.zephyr.com/v1/products?per_page=100&page=${page}`,
    {
      headers: { 'Authorization': `Bearer ${apiKey}` }
    }
  );

  const data = await response.json();
  products.push(...data.data);

  if (data.meta.page < data.meta.total_pages) {
    return fetchAllProducts(page + 1, products);
  }

  return products;
}
```

### 2. Caching

Cache product data aggressively to reduce API calls:

```javascript
// Cache products for 5 minutes
const productCache = new Map();

async function getProduct(productId) {
  const cached = productCache.get(productId);

  if (cached && Date.now() - cached.timestamp < 300000) {
    return cached.data;
  }

  const response = await fetch(
    `https://api.zephyr.com/v1/products/${productId}`,
    {
      headers: { 'Authorization': `Bearer ${apiKey}` }
    }
  );

  const data = await response.json();

  productCache.set(productId, {
    data,
    timestamp: Date.now()
  });

  return data;
}
```

<Callout type="info">
  **Tip:** Product data changes infrequently (except for inventory). Cache product responses on your edge CDN to reduce latency. Only invalidate cache on product updates.
</Callout>

### 3. Webhooks for Synchronization

Subscribe to product webhooks to keep your local database synchronized:

```javascript
// Handle product updates
app.post('/webhooks/products', (req, res) => {
  const event = req.headers['x-webhook-event'];

  switch (event) {
    case 'product.created':
      handleProductCreated(req.body.data);
      break;
    case 'product.updated':
      handleProductUpdated(req.body.data);
      break;
    case 'product.deleted':
      handleProductDeleted(req.body.data);
      break;
    case 'product.inventory_low':
      handleLowInventory(req.body.data);
      break;
  }

  res.status(200).send('OK');
});

function handleProductUpdated(product) {
  // Update local database
  db.products.updateOne(
    { id: product.id },
    { $set: product },
    { upsert: true }
  );

  // Invalidate cache
  productCache.delete(product.id);
}
```

### 6. Handle Variants Correctly

Always display variant-specific information to customers:

```javascript
function displayProduct(product) {
  if (product.variants.length > 0) {
    // Product has variants - show variant selector
    return (
      <ProductVariantSelector
        variants={product.variants}
        onSelectVariant={(variant) => addToCart(variant)}
      />
    );
  } else {
    // Product has no variants - show product
    return (
      <ProductDisplay
        product={product}
        onAddToCart={() => addToCart(product)}
      />
    );
  }
}
```

## Error Handling

### Common Errors

| Error Code | Description | Solution |
|------------|-------------|----------|
| `product_not_found` | Product doesn't exist | Verify product ID |
| `invalid_sku` | SKU already exists | Use unique SKU |
| `invalid_price` | Invalid price value | Check price format |
| `insufficient_inventory` | Not enough inventory | Update inventory or change policy |
| `variant_required` | Product requires variants | Create at least one variant |

**Example Error Response:**

```json
{
  "error": {
    "type": "validation_error",
    "message": "SKU already exists",
    "code": "invalid_sku",
    "details": {
      "field": "sku",
      "value": "SH-2024-BLK"
    },
    "request_id": "req_abc123xyz",
    "timestamp": "2026-05-10T15:30:00Z"
  }
}
```
